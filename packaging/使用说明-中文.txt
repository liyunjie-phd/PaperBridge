PaperBridge 0.2.0 Windows 完整版使用说明
====================================

一、版本说明

收到 ZIP 后请先完整解压到一个普通文件夹，再运行其中的程序。
不要直接在压缩软件的预览窗口中启动 EXE。

双击 PaperBridge-Setup.exe 完成安装。该版本包含内置 Tectonic 排版组件，
电脑没有安装 TeX Live 或 MiKTeX 也可以编译论文。

如果电脑已经安装 TeX Live 或 MiKTeX，并且可以使用 latexmk，PaperBridge 会优先使用本机 LaTeX；
否则自动使用内置 Tectonic。安装后会创建桌面和开始菜单快捷方式，
设置保存在 Windows 用户应用数据目录中，论文保存在“文档\PaperBridge Projects”。

二、Windows 安全提示

当前版本没有购买商业代码签名证书。
Windows 第一次运行时可能显示“Windows 已保护你的电脑”。

确认文件来自发送者并完成 SHA-256 校验后，可以：

1. 点击“更多信息”。
2. 确认应用名称为 PaperBridge。
3. 点击“仍要运行”。

如果学校或公司的电脑没有“仍要运行”按钮，说明安全策略禁止未签名软件，
请改用个人 Windows 10/11 电脑测试，不要关闭单位电脑的安全策略。

三、首次配置

1. 启动 PaperBridge。
2. 选择论文来源：Overleaf 链接、GitHub/GitLab HTTPS 仓库、Overleaf ZIP，或本地 LaTeX 文件夹。
3. 如果使用 Overleaf 链接，请填写自己的 Overleaf Git Token。PaperBridge 会在拉取和推送时自动完成认证，不需要输入 Git 密码。
4. 在 AI 设置中选择 API 服务类型和模型，再填写自己的 API 地址与 API Key。DeepSeek 可以直接从下拉框选择 V4 Flash 或 V4 Pro。
5. 不要把自己的 API Key、Overleaf Token 或 PaperBridge-Data 文件夹发给别人。

如果 Overleaf 显示“no git access”，常见原因是项目链接错误、项目不存在、账户没有项目权限，或项目所有者没有开通 Overleaf Git 高级功能。该功能通常需要个人/团队订阅或学校的 Overleaf Commons 权限。没有权限时，请从 Overleaf 下载 ZIP 后导入。

GitHub/GitLab 可以直接选择“Git 仓库”进行克隆。选择 ZIP 或本地文件夹时，也可以勾选“同时连接 GitHub / GitLab 仓库”。建议使用新建的空仓库；导入不会自动上传，确认无误后再点击顶部“推送”。私有仓库需要 Git 用户名和 Personal Access Token，目前只支持 HTTPS 地址。

应用支持 OpenAI 兼容接口、Anthropic 和 Gemini。使用第三方 API 前，
请确认论文内容可以发送给该服务，并注意 API 费用和数据隐私要求。

四、主要功能

- 左侧修改中文，右侧更新对应英文 LaTeX。
- 每个段落右上角的加号可以在当前段落之前或之后新增正文；输入中文后会自动生成对应英文。垃圾桶按钮用于删除不再需要的段落。
- AI 输出包含危险 LaTeX 命令时会被直接阻止；新增其他 LaTeX 命令时会显示命令清单并要求确认。
- 以英文 LaTeX 为排版依据并实时编译 PDF。
- PDF 支持缩放、拖动和下载。
- 可以上传 Word、PDF、TeX 或 ZIP 格式模板，分析并迁移论文格式。
- 可以在完成修改后进行英文语法和全文连贯性检查。
- 导入 ZIP 或本地文件夹时，会自动跳过导言区、作者信息、宏定义、公式、图表、算法代码和参考文献。
- 可以从 GitHub/GitLab 仓库拉取和推送；对于没有共同历史的非空远端，PaperBridge 会拒绝覆盖。
- 普通 Git 仓库首次推送前会显示完整文件清单，默认只选择 TeX、参考文献、样式文件和图像。

中文工作稿会按项目依次保存，并在 PaperBridge 数据目录中保留一份最近备份。
备份不会写入论文文件夹或 Git 仓库。PaperBridge 也会拒绝编辑指向项目目录之外的符号链接。

左侧“文档”标题旁的文件夹加号可以添加或切换论文。顶部“拉取”和“推送”用于更新当前 Overleaf 项目。

格式迁移和 AI 修改前请保留 Overleaf 或论文文件夹的独立备份。

五、校验下载文件

同一目录中的 SHA256.txt 记录了安装程序的校验值。

简单方法：

1. 在当前文件夹空白处按住 Shift 并单击鼠标右键。
2. 选择“在此处打开 PowerShell 窗口”或“在终端中打开”。
3. 输入：

   Get-FileHash .\PaperBridge-Setup.exe -Algorithm SHA256

4. 将显示的 Hash 与 SHA256.txt 对照，必须完全一致。

六、卸载和清理

从 Windows“设置 > 应用 > 已安装的应用”卸载 PaperBridge。
卸载程序会询问是否同时删除本地数据：

- 选择“否”：保留论文项目、API 配置和本地设置，以后重新安装仍可继续使用。
- 选择“是”：永久删除“文档\PaperBridge Projects”中的全部项目，以及 PaperBridge 的设置和缓存。

删除无法撤销。选择“是”之前，请先备份仍需保留的论文和附件。
